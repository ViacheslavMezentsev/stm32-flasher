# Инструкция по использованию stm32-flasher

## Назначение

`stm32-flasher` - это Windows-скрипт для прошивки STM32 через ST-Link без отдельной установки проекта или сложной подготовки рабочего места. Основной сценарий - положить `flash.cmd` рядом с `*.hex` и запустить его двойным щелчком или передать файл в командной строке.

Скрипт старается оставаться самодостаточным:
- работает как один `flash.cmd`;
- использует `STM32CubeProgrammer`, если он установлен;
- иначе может скачать и использовать встроенный `OpenOCD`;
- при необходимости использует `st-info` для определения типа подключённого STM32;
- если к ПК подключено несколько ST-Link, позволяет выбрать нужный;
- сохраняет лог и HTML-отчёт последней прошивки.

## Что ожидать от поведения

### Обычный сценарий

1. Пользователь кладёт `flash.cmd` рядом с `*.hex`.
2. Подключает ST-Link к ПК и плате.
3. Запускает `flash.cmd`.

Дальше поведение зависит от содержимого папки и параметров запуска:
- если рядом только один `*.hex`, скрипт сразу начинает прошивку;
- если `*.hex` несколько, скрипт предложит выбрать нужный;
- если файл `*.hex` передан на скрипт мышкой или аргументом командной строки, он будет использован напрямую;
- если найден `STM32CubeProgrammer`, в прямом сценарии он выбирается автоматически как предпочтительный движок;
- при успешной прошивке отчёт сохраняется, но не открывается автоматически;
- при ошибке отчёт сохраняется и открывается автоматически.

### Что сохраняется после запуска

| Файл/папка | Назначение |
|---|---|
| `.tools\` | Локальная папка инструментов, в том числе OpenOCD |
| `.flash_engine` | Запомненный выбор движка прошивки |
| `.openocd_target` | Запомненный `target/*.cfg` для OpenOCD |
| `.stlink_serial` | Запомненный выбор конкретного ST-Link |
| `flash_log.txt` | Лог последней прошивки |
| `report.html` | HTML-отчёт последней прошивки |

## Требования

- Windows 10 / 11;
- PowerShell 5.1 или новее;
- подключённый ST-Link;
- драйверы ST-Link;
- интернет при первом запуске, если потребуется скачать OpenOCD или полный набор `stlink` tools.

## Способы запуска

### 1. Двойной щелчок по `flash.cmd`

Подходит для обычного использования в Проводнике Windows.

Рекомендуемая структура:

```text
flash.cmd
mcu_lts_board.hex
```

Если `*.hex` один, прошивка стартует сразу.

### 2. Drag-and-drop

Можно перетащить `*.hex` на `flash.cmd`. В этом случае скрипт использует переданный файл как цель прошивки.

### 3. Командная строка

Поддерживаются как позиционный аргумент, так и именованные параметры.

Примеры:

```powershell
.\flash.cmd .\mcu_lts_board.hex
.\flash.cmd -HexFile .\mcu_lts_board.hex
.\flash.cmd -HexFile .\mcu_lts_board.hex -Lang ru
.\flash.cmd -HexFile .\mcu_lts_board.hex -Engine OPENOCD
.\flash.cmd -HexFile .\mcu_lts_board.hex -Target target/stm32g4x.cfg
.\flash.cmd -HexFile .\mcu_lts_board.hex -Serial 066AFF535548877187165246
.\flash.cmd -HexFile .\mcu_lts_board.hex -DryRun
.\flash.cmd -HexFile .\mcu_lts_board.hex -Silent
```

## Параметры

| Параметр | Назначение |
|---|---|
| `-HexFile <path>` | Явно задаёт путь к `*.hex` |
| `-Lang <ru\|en>` | Принудительно выбирает язык интерфейса и отчёта |
| `-Engine <OPENOCD\|path>` | Выбирает движок прошивки |
| `-Target <target/...cfg>` | Явно задаёт target-конфиг OpenOCD |
| `-Serial <id>` | Явно выбирает конкретный ST-Link по его serial |
| `-Silent` | Уменьшает количество вывода в консоль |
| `-DryRun` | Проверяет сценарий без реальной прошивки |

## Как выбирается движок

Скрипт поддерживает два движка:
- `STM32CubeProgrammer`;
- `OpenOCD`.

Логика выбора:
- если движок задан явно параметром `-Engine`, используется он;
- если ранее был сохранён выбор в `.flash_engine`, он будет использован;
- если запуск идёт по прямому сценарию без лишних вопросов, скрипт предпочитает `STM32CubeProgrammer`, если он найден;
- иначе предлагается выбор из доступных вариантов.

## Как выбирается программатор

Если к ПК подключён один ST-Link, он используется автоматически.

Если подключено несколько ST-Link:
- можно сразу указать нужный параметром `-Serial`;
- если параметр не передан, скрипт покажет меню выбора;
- рядом с serial по возможности показывается распознанный тип МК;
- выбранный serial сохраняется в `.stlink_serial` для следующего запуска.

Этот же выбранный программатор затем используется и для `OpenOCD`, и для `STM32CubeProgrammer`.

## Как выбирается target для OpenOCD

Если используется OpenOCD, скрипт пытается определить `target/*.cfg` последовательно:

1. из параметра `-Target`;
2. из сохранённого `.openocd_target`;
3. по данным `st-info`, если утилита доступна и удалось распознать подключённый МК;
4. по данным `*.ioc`, если такой файл найден рядом;
5. по имени `*.hex`;
6. вручную через ввод пользователя.

Если указанный или сохранённый `target/*.cfg` не существует внутри `OpenOCD scripts`, скрипт не использует его молча, а сообщает о проблеме.

### Как используется `st-info`

Для OpenOCD скрипт старается определить семейство STM32 по подключённому устройству через `st-info --probe`.

Сначала он ищет `st-info`:
- в `PATH`;
- в стандартных путях установки;
- в локальной `.tools\stlink`;
- при необходимости скачивает полный набор `stlink` tools из GitHub Releases.

Если `chipid` успешно найден, скрипт сопоставляет его с подходящим `target/*.cfg` для OpenOCD и пишет диагностическое сообщение в консоль.

## Отчёт и лог

После каждого запуска формируются:
- `flash_log.txt`;
- `report.html`.

HTML-отчёт содержит:
- результат прошивки;
- состояние этапов;
- сведения о хосте;
- сведения о пользователе и машине;
- сведения о ST-Link;
- сведения о микроконтроллере;
- лог утилиты;
- локальное время и UTC-время формирования отчёта.

Поведение по умолчанию:
- при успехе отчёт только сохраняется;
- при ошибке отчёт сохраняется и открывается автоматически.

## Локализация

Поддерживаются два языка:
- русский;
- английский.

По умолчанию язык выбирается по системной локали. При необходимости можно явно указать язык параметром:

```powershell
.\flash.cmd -HexFile .\mcu_lts_board.hex -Lang en
```

## Dry Run

Режим `-DryRun` полезен для проверки сценария без записи во Flash. Он позволяет:
- проверить выбор файла;
- проверить выбор движка;
- проверить выбор конкретного ST-Link;
- проверить определение target;
- получить лог и HTML-отчёт без реальной прошивки платы.

## Ограничения

- скрипт рассчитан на Windows;
- основной сценарий - прошивка STM32 через ST-Link;
- логика отчёта ориентирована на `STM32CubeProgrammer` и `OpenOCD`;
- `report.html` и `flash_log.txt` всегда отражают только последний запуск.

## Рекомендуемый сценарий для пользователя

Если нужен самый простой способ работы:

1. положить `flash.cmd` рядом с единственным `*.hex`;
2. подключить ST-Link;
3. запустить `flash.cmd`;
4. при необходимости открыть `report.html` вручную.
